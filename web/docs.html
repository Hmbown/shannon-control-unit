Profiles, API Reference, and Validation Guidelines

Anomaly Detection API

Detection Profiles

drug_response: window_size=16384, numeric_preprocess=zscore_quant8, sensitivity=1.25

network_logs: window_size=4096, anomaly_threshold=1.2, sensitivity=1.25

Quick Start

Start the API server:

python -m uvicorn api.main:app --host 0.0.0.0 --port 8000

Detect anomalies:

curl -s -X POST http://localhost:8000/v1/detect \
  -H "Authorization: Bearer shannon_dev_key_001" \
  -H "Content-Type: application/json" \
  -d '{
    "data":"SGVsbG8gV29ybGQh",
    "domain":"biomedical",
    "profile":"drug_response",
    "window_size":16384,
    "numeric_preprocess":"zscore_quant8",
    "sensitivity":1.25,
    "anomaly_threshold":1.2
  }'

Get configuration:

curl -s http://localhost:8000/v1/config | jq .

Endpoints

POST /v1/detect - Detect anomalies in data

GET /v1/config - Get current configuration

GET /v1/health - Health check endpoint

Planned Endpoints:

POST /v1/llm/chat/completions - OpenAI-compatible chat completions

POST /v1/llm/embeddings - Generate embeddings with anomaly scoring

Model Variants:

  • shannon-1b - Edge deployment, 1B parameters
  • shannon-7b - Balanced performance, 7B parameters
  • shannon-70b - Enterprise-grade, 70B parameters

Unique Features:

  • Built-in anomaly detection for all responses
  • MDL (Minimum Description Length) optimization
  • Compression-aware training for better efficiency
  • Real-time security scoring

Example Usage (When Available)

# Chat completions with anomaly detection
curl -X POST https://api.shannonlabs.dev/v1/llm/chat/completions \
  -H "Authorization: Bearer your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "shannon-7b",
    "messages": [
      {"role": "user", "content": "Explain quantum computing"}
    ],
    "anomaly_detection": true,
    "max_tokens": 1000
  }'

Response will include anomaly scores and confidence metrics.
